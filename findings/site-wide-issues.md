# Site-Wide Accessibility Issues

Issues generated by the Squarespace template that appear on **every** page of littleflowerlambs.com. Documented once here rather than repeated in each page's findings file. Individual page findings reference this file by name.

**Standard:** WCAG 2.1
**First observed:** Homepage (page 1 of 10)
**Confirmation status:** Confirmed on homepage; being verified page-by-page as the audit proceeds. Check marks added below as each page confirms the element is present and behaves identically.

---

## 1. Announcement bar — low-contrast text

- **What:** The top announcement bar renders text at very low contrast against its solid background. Documented example: the "Make your bid here" text measured **1.15:1** (WCAG AA requires 4.5:1 for normal text).
- **Why site-wide:** The announcement bar is a template header element rendered on every page, not page content.
- **Fix path:** Darken the foreground text (or lighten/darken the bar background) in Squarespace's style editor. A single style change resolves it everywhere the bar appears.
- **WCAG:** 1.4.3 Contrast (Minimum)
- **Bucket:** Fixed candidate (solid background, measurable, editable in the style panel)
- **Evidence:** `screenshots/homepage-wave-contrast-announcementbar.png`

## 2. Footer background video — keyboard-unreachable footer

- **What:** After tabbing into the footer's embedded video controls, focus bypasses all subsequent footer content (footer navigation, social links) and returns directly to browser chrome. Not a keyboard trap — focus escapes — but the footer region is skipped entirely, so its links can't be reached by keyboard.
- **Why site-wide:** The footer (including the background video) is a template element rendered on every page.
- **Fix path:** Uncertain at contractor access level. May require code injection to correct the tab sequence, or may be locked at the platform/template level. Flag for investigation, not committed as fixable yet.
- **WCAG:** 2.4.3 Focus Order (footer content skipped); relates to 2.1.1 Keyboard
- **Bucket:** Identified But Not Fixed (pending confirmation of whether it's reachable at contractor access)

## 3. Redundant links (candidate — confirm site-wide)

- **What:** The same destination is linked in adjacent elements (e.g., logo and text both linking to home). WAVE flags this as an alert, not an error. Minor.
- **Why possibly site-wide:** The header logo/text link pattern is template-generated and likely repeats on every page. **Not yet confirmed** as identical across pages — verify before treating as fully site-wide.
- **Fix path:** Often not correctable at contractor level (template markup). Low priority regardless.
- **WCAG:** Best-practice alert (not a strict failure)
